# PBS 檔案翻譯指南

本目錄包含從 `PBS/` 目錄複製的所有遊戲資料檔案，用於中文化翻譯工作。

## 📋 檔案總覽

共 28 個 .txt 檔案需要處理。

---

## 🎯 核心檔案（優先翻譯）

### 1. **pokemon.txt** - 寶可夢資料
**用途**: 定義所有寶可夢的基礎資料和圖鑑描述

**需要翻譯的欄位**:
- `Name` - 寶可夢名稱（例：Bulbasaur → 妙蛙種子）
- `Kind` - 分類（例：Seed → 種子寶可夢）
- `Pokedex` - 圖鑑描述（整段文字）

**不需要翻譯的欄位**:
- `InternalName` - 內部程式碼名稱（保持英文大寫）
- `Type1`, `Type2` - 屬性代碼（保持英文大寫）
- `BaseStats`, `GenderRate`, `GrowthRate` - 數值和遊戲參數
- `Abilities`, `HiddenAbility` - 特性代碼（保持英文大寫）
- `Moves`, `EggMoves` - 招式代碼（保持英文大寫）

**翻譯注意事項**:
- 寶可夢名稱需參考官方中文譯名
- 圖鑑描述需保持原文語氣和風格
- 分類名稱通常為「XX寶可夢」格式

---

### 2. **moves.txt** - 招式資料
**用途**: 定義所有招式的名稱和說明

**檔案格式**: `編號,內部名稱,顯示名稱,功能碼,威力,屬性,類別,命中率,PP,附加效果機率,目標,優先度,旗標,說明`

**需要翻譯的欄位**:
- 欄位3: `顯示名稱` - 招式的顯示名稱（例：Megacuerno → 超級角擊）
- 欄位13: `說明` - 招式效果描述（引號內的文字）

**不需要翻譯的欄位**:
- 欄位1: 編號
- 欄位2: 內部名稱（保持英文大寫）
- 欄位4-12: 數值參數（功能碼、威力、屬性、類別等）

**翻譯注意事項**:
- 招式名稱需參考官方中文譯名
- 招式說明需簡潔清楚，保持遊戲風格
- 注意特殊符號和引號的完整性

---

### 3. **items.txt** - 道具資料
**用途**: 定義所有道具的名稱和效果

**檔案格式**: `編號,內部名稱,顯示名稱,複數名稱,口袋,價格,說明,使用場合,戰鬥使用,重要性`

**需要翻譯的欄位**:
- 欄位3: `顯示名稱` - 道具的顯示名稱（例：Repel → 除蟲噴霧）
- 欄位4: `複數名稱` - 複數形式的名稱
- 欄位7: `說明` - 道具效果描述（引號內的文字）

**不需要翻譯的欄位**:
- 欄位1: 編號
- 欄位2: 內部名稱（保持英文大寫）
- 欄位5-10: 數值參數（口袋、價格等）

**翻譯注意事項**:
- 道具名稱需參考官方中文譯名
- 說明文字需清楚說明道具用途和效果
- 西班牙語中的複數形式在中文中通常保持不變

---

### 4. **trainers.txt** - 訓練師資料
**用途**: 定義訓練師的隊伍和對話

**檔案格式**:
```
訓練師類型
訓練師名稱
隊伍數量
寶可夢資料...
```

**需要翻譯的欄位**:
- 訓練師類型名稱（例：Mochilero → 背包客）
- 訓練師名稱（人名可選擇性翻譯或音譯）

**不需要翻譯的欄位**:
- 寶可夢內部名稱（AZURILL、FLETCHLING 等）
- 招式內部名稱（TACKLE、TAILWHIP 等）
- 等級數字

**翻譯注意事項**:
- 訓練師類型需參考官方中文譯名（如「背包客」、「警察」等）
- 訓練師名字可依風格選擇音譯或創作符合角色的中文名
- 保持檔案格式和縮排

## 📝 通用翻譯規範

### 編碼要求
- **必須使用 UTF-8 編碼**
- 保存時確認無 BOM (Byte Order Mark)
- 使用 UTF-8 without BOM 格式

### 格式規範
1. **保持檔案結構**: 不改變行數、欄位數量和分隔符
2. **CSV 格式**: 使用逗號 (,) 分隔欄位
3. **引號處理**: 如果文字包含逗號，需用引號包圍
4. **換行符號**: 保持原始檔案的換行格式

### 內部名稱規範
- 所有 `InternalName` 欄位**必須保持英文大寫**
- 不可翻譯任何內部代碼（如 `BULBASAUR`, `TACKLE`, `GRASS` 等）
- 這些代碼用於程式邏輯，改動會導致遊戲錯誤

### 翻譯風格
1. **參考官方**: 優先使用寶可夢官方中文譯名
2. **保持一致**: 同一術語在所有檔案中使用相同翻譯
3. **簡潔清晰**: 遊戲文字需簡潔，避免冗長
4. **符合角色**: 對話和描述需符合角色性格和遊戲氛圍

## ⚠️ 常見錯誤

1. **編碼錯誤**: 使用非 UTF-8 編碼導致亂碼
2. **格式破壞**: 遺漏逗號或引號導致解析錯誤
3. **內部名稱翻譯**: 將 InternalName 翻譯成中文導致遊戲崩潰
4. **不一致翻譯**: 同一術語在不同檔案使用不同譯名
5. **遺漏欄位**: 翻譯時遺漏或刪除某些欄位
